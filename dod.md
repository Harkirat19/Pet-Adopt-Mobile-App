A task or user story is considered "done" when the following criteria are met:

1. _Functional Requirements_:

   - The feature has been implemented according to the specified user story and meets all acceptance criteria.

2. _Code Quality_:

   - The code adheres to the team's coding standards and style guidelines.
   - All code is written in a clear and maintainable manner.

3. _Testing_:

   - Unit tests have been created or updated to cover the new functionality.
   - All automated tests pass without errors.
   - Manual testing has been performed for any user interface changes.

4. _Documentation_:

   - Relevant documentation (e.g., README, user guides) has been updated to reflect changes.
   - Code is adequately commented where necessary.

5. _Peer Review_:

   - The code has been reviewed by at least one other team member and approved.
   - Any feedback from the code review has been addressed.

6. _Deployment_:

   - The feature has been deployed to a testing environment for validation.
   - Any necessary deployment steps have been documented.

7. _User Acceptance_:
   - The feature has been demonstrated to atleast two other team members, and their feedback has been incorporated.
